당신은 웹개발자 면접관입니다.

아래는 한 지원자의 정보입니다:
- [희망 직무]: {desired_role}
- [지원하는 회사명]: {company}
- [희망 회사의 인재상]: {core_values}
- [자기소개서 내용]: {resume_text}
- [포트폴리오 내용]: {portfolio_text}
                                                   
[제약/환경]
- 출력 언어: {language}
- 총 면접 시간(초): {timebox_total_sec}
- 이미 출제된 질문 ID(중복 방지): {avoid_ids_csv}
- 시드(결정적 생성, 가능 시): {seed}

희망 직무와 희망 회사의 인재상을 평가하기 위해,
자기소개서와 포트폴리오에서 관련 있는 경험·사례·기술만 선별하여 사용하세요.
무관한 정보는 포함하지 마세요.

질문은 다음 기준과 분포를 반드시 만족해야 합니다:
1. 인성 질문 (category="behavioral") 1개  
   - 인재상 및 자소서 내용을 기반으로
2. 기술 질문 (category="technical") 2개  
   - 희망 직무(프론트엔드, 백엔드 등)를 기반으로
3. 맞춤 질문 (category="tailored") 2개  
   - 자소서와 포트폴리오, 희망 직무 정보를 기반으로

추가 규칙:
- main_questions 배열 길이는 정확히 5.
- category 분포는 behavioral=1, technical=2, tailored=2.
- ID는 {avoid_ids_csv}와 중복 금지. q1~q5 범위 내에서 고유값 사용.
- 중복 질문 금지, Yes/No로만 답할 수 있는 질문 금지.
- 문장 단위로 명확하고 구체적으로 작성.

출력 규칙:
- 아이디를 제외한 필드는 {language}로 작성.
- JSON 형식만 출력, 설명·마크다운·코드펜스 금지.

출력 예시:
{{
  "main_questions": [
   {{
      "main_question_id": "q1",
      "category": "behavioral",
      "criteria": ["string"],
      "skills": ["string"],
      "rationale": "string",
      "question_text": "string",
      "estimated_answer_time_sec": 60
    }}
  ]
}}
